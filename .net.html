  3. 使用Parallel类：Parallel类是.NET Framework提供的一个类，可以用来并行执行多个操作。
  4. 使用async/await关键字：async/await关键字可以让你以同步的方式写异步代码，这样就可以实现多线程了。

☘️前台线程和后台线程有什么差别？

前台线程是指用户可见的线程，它们可以接收用户的输入，并且处理用户的请求。后台线程是指在后台运行的线程，它们不可见，但它们可以完成一些重要的任务，例如清理缓存，垃圾回收，数据库操作等。

应用程序必须运行完所有的前台线程才可以退出；

而对于后台线程，应用程序则可以不考虑其是否已经运行完毕而直接退出，所有的后台线程在应用程序退出时都会自动结束。

☘️C#的同步异步如何实现？

C#通过提供异步方法和异步委托来实现异步操作。

异步方法使用async和await关键字来声明，它们可以让程序在后台运行而不会阻塞前台的执行流程。异步委托使用BeginInvoke和EndInvoke方法来实现，它们可以让程序在后台运行而不会阻塞主线程的执行流程。

  • C#中使用多线程实现异步
  • 用C#语言提供的多线程实现异步,这里使用Task来实现,主要使用await 和async.

Task类

相比于Thread类，Task类为控制线程提供了更大的灵活性。

  • Task类可以获取线程的返回值
  • 可以定义连续的任务：在一个任务结束结束后开启下一个任务
  • 可以在层次结构中安排任务，在父任务中可以创建子任务 这样就创建了一种依赖关系，如果父任务被取消，子任务也随之取消

任务Task和线程Thread的区别：

  • 任务是架构在线程之上的 也就是说任务最终还是要抛给线程去执行。
  • 任务跟线程不是一对一的关系 比如开10个任务并不是说会开10个线程，这一点任务有点类似线程池，但是任务相比线程池有很小的开销和精确的控制。
  • Task和Thread一样，位于System.Threading命名空间下。

☘️线程池相对于基础的线程有什么优缺点？

多线程的缺点：

1、线程开的越多，内存占用越大 2、协调和管理代码的难度加大，需要CPU时间跟踪线程 3、线程之间对资源的共享可能会产生可不遇知的问题

合理利用线程池能够带来三个好处：

  • 降低资源消耗。通过重复利用已创建的线程降低线程创建和销毁造成的消耗。
  • 提高响应速度。当任务到达时，任务可以不需要等到线程创建就能立即执行。
  • 提高线程的可管理性。线程是稀缺资源，使用线程池可以进行统一的分配、调优和监控。

☘️设置线程池的最大进程数是什么目的？

线程池的最大进程数可以帮助我们限制系统的资源消耗，以此来避免创建过多的线程而导致性能下降。

使用线程池可以有效地管理系统资源，从而减少线程创建所需要的时间，提高程序的效率。

☘️多个线程之间访问同一个数据时如何保证数据的线程安全？多线程并发安全？

  1. 使用同步机制，例如锁或信号量，来确保每一次只有一个线程访问数据。
  2. 合理的设置线程优先级，确保线程之间的公平性，避免出现优先级反转。
  3. 使用原子操作，保证操作的原子性，避免出现数据不一致的情况。
  4. 通过对数据实施复制和缓存，避免对共享数据进行读写操作，从而保证线程间的安全性。

☘️C#中两个线程之间如何实现通信的机制？

C#中提供了几种线程间通信的机制：

1）事件：使用事件可以使一个线程等待其他线程完成某项任务。

2）信号量：可以用来限制线程访问某个特定资源的数量。

3）信号：用来向线程发送信号，告诉其他线程何时可以开始执行。

4）同步：用来控制多个线程之间的执行顺序，以避免竞争条件的出现。

5）管道和套接字：使用这些技术可以在不同的进程之间传递数据。

C#中可以使用如下几种机制来实现线程之间的通信：

  1. 信号量：信号量是最常用的线程通信机制，它用于限制同时访问特定资源的线程数量。信号量是一个计数器，可以在多个线程之间共享，当一个线程需要访问某个资源时，它会检查信号量的计数器，如果计数器大于0，则减少该计数器，表示可以访问该资源；如果计数器等于0，则表示该资源已经被其他线程占用，当前线程应该等待。
  2. 事件：使用事件来实现线程间的通信也是一个常用的方法。当一个线程需要通知另一个线程做某件事时，它会触发一个事件，这样另一个线程就可以监听这个事件，然后做出相应的反应。
  3. 共享内存：使用共享内存也是实现线程间通信的一个常用方法。在共享内存中，多个线程可以读写同一块内存，这样就可以实现两个或多个线程之间的通信。
  4. 消息队列：使用消息队列也是实现线程间通信的一个常用方法。消息队列是一个可以在多个进程之间共享的数据存储区，多个进程可以向其中发送消息，而其他进程也可以从中接收消息。

☘️C#中的委托？

C#中的委托是一种类型安全的函数指针，它允许将方法作为参数传递给另一个方法。它是一种将方法当作另一个方法的参数，以便在运行时调用该方法。它允许您将代码段作为参数传递给另一个方法，以实现回调。

Q: 一般什么时候用到委托？ 委托通常在以下情况下使用：

  1. 当某个对象需要将其工作委托给另一个对象时。
  2. 当某个对象不想处理某些特定的事件时，可以将事件处理委托给另一个对象。
  3. 当某个对象想要使用另一个对象的功能，但是又不想知道另一个对象的实现细节时，可以使用委托。

委托是一种类型安全的函数指针，它允许把函数当做参数来传递，从而实现回调函数和事件处理。使用委托可以实现面向对象编程中重要的回调机制，常见的应用场景有：

  • 异步操作，将委托定义为异步操作完成时的回调方法；
  • 事件处理，将委托定义为事件发生时的响应方法；
  • 回调函数，将委托定义为某种特定条件发生时的回调方法；
  • LINQ 查询，将委托定义为查询条件；
  • 其他操作，如自定义排序、合并数据等；

☘️C#中的值类型和引用类型

值类型：在内存中存储的是变量的实际值，例如int、float、char等。

引用类型：在内存中存储的是对象的地址，例如string、object、class等。

值类型：传递的是参数的副本，不影响原始值

引用类型做值参数：实参和形参的引用 是两个东西，但两个引用指向堆中同一个数据，因此在方法中改变形参会影响实参的值

  • 引用类型变量的赋值只复制对对象的引用，而不复制对象本身。而将一个值类型变量赋给另一个值类型变量时，将复制包含的值；

  • 引用类型可以派生出新的类型，而值类型不能；

  • 引用类型可以包含null值，值类型不能（可空类型功能允许将 null 赋给值类型，但是调用时会抱错：使用为赋值类型）；

  • 值类型总是分配在它声明的地方，作为字段时，跟随其所属的变量（实例）存储在堆上，作为局部变量时，存储在栈上。引用类型存储在堆中。类型实例化的时候，会在堆中开辟一部分空间存储类的实例，类实例的引用（指针）还是存储在栈中。

☘️C#中的装箱和拆箱？

装箱就是隐式的将一个值型转换为引用型对象。

拆箱就是将一个引用型对象转换成任意值型。

为什么需要装箱（为何要将值类型转为引用类型？）

  • 一种最普通的场景是，调用一个含类型为Object的参数的方法，该Object可支持任意为型，以便通用。当你需要将一个值类型(如Int32)传入时，需要装箱。
  • 另一种用法是，一个非泛型的容器，同样是为了保证通用，而将元素类型定义为Object。于是，要将值类型数据加入容器时，需要装箱。

如何更改已装箱的对象 ？

  • 对于已装箱的对象，因为无法直接调用其指定方法，所以必须先拆箱，再调用方法，但再次拆箱，会生成新的栈实例，而无法修改装箱对象。

WPF路由事件是WPF中比较特殊的事件类型，它可以沿着元素树从根元素到叶元素依次触发。 WPF路由事件主要包括三种类型：Tunneling、Bubbling和Direct，它们之间的区别如下：

Direct：直接事件，只在当前元素上触发，不会向上传播或者向下传播。

22.谈谈final, finally, finalize的区别。

finalize：方法名。Java 技术允许使用 finalize() 方法在垃圾收集器将对象从内存中清除出去之前做必要的清理工作。这个方法是由垃圾收集器在确定这个对象没有被引用时对这个对象调用的。它是在 Object 类中定义的，因此所有的类都继承了它。子类覆盖 finalize() 方法以整理系统资源或者执行其他清理工作。finalize()方法是在垃圾收集器删除对象之前对这个对象调用的。

引用传递：在调用函数时，传入的是实参的引用，函数对实参的操作会影响原来的实参。

可以通过 ref 和 out 来决定参数是否按照引用传递。

C#中参数传递 ref 与 out 的区别？

（ 1 ） ref 指定的参数在函数调用时必须先初始化，而 out 不用 （ 2 ） out 指定的参数在进入函数时会清空自己，因此必须在函数内部进行初始化赋值操作，而 ref 不用 总结： ref 可以把值传到方法里，也可以把值传到方法外； out 只可以把值传到方法外 注意： string 作为特殊的引用类型，其操作是与值类型看齐的，若要将方法内对形参赋值后的结果传递出来，需要加上ref 或 out 关键字。
